## 结构目录说明

```
bin :可执行文件目录 用于开发
build: 部署目录
common: 通用组件  PS(根据开发规范增加通用组件都需要写test)
  - global: 为全局变量定义
  - msg: 为全局消息定义
  - util: 为通用工具定义
data: 程序数据目录(mysql数据，td数据，nginx数据等)
deploy: 部署目录(包含各种中间配置，程序配置)
service: 代码主体
test: 测试代码
makefile: 编译脚本
.gitignore: git忽略文件
readme.md: 说明文档
```
